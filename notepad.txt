ClASS 02 & Class 03: Introduction to web development | Computer networking basics | WED & THU (✔) 
Web = internet par kisi bhi audio, video, file, ya PDF ki interconnectivity ke liye use hota hai.
Development  = building ya construct karna (websites ya web apps ko design aur develop karna).
Website = static in nature, iska content fixed hota hai, jo har user ke liye same dikhta hai. Example: Portfolio websites, basic informational sites.
Web app = dynamic in nature, iska content user ke interaction ke hisaab se change hota hai. Example: Facebook, Gmail, e-commerce websites.
Front end = jo user ke saamne dikh raha hai (UI/UX). kch languages/tools ye hain: HTML, CSS, JavaScript, React, Angular.
Backend = jo system ke piche chal raha hota hai (logic, server-side). kch languages/tools ye hain: Node.js, Python (Django/Flask), PHP, Java.
Database = Data ko store aur manage karne ka system. MySQL, MongoDB, PostgreSQL.
Fullstack = jo ye saari cheezen dekhta ho frontend, backend aur database.
How can you take best out of this course = notes banaanay hain hr class ke khud se aur hr module ke end main test hoga
Network = devices ka ek doosre ke saath connected system. 1. wired (cables ka use hota hai) aur 2. wireless (Wi-Fi, Bluetooth, etc).
LAN (local Area Network) = local network hota hai, jaise ghar ka Wi-Fi ya office ka system. Ek limited area tak restricted hota hai.
MAN (metropolitan area network) = city-to-city ka network. Example: Islamabad se Peshawar. Do ya zyada LANs ko connect karta hai.
WAN (wide area network) = country-level ya global network. Example: Internet.
Internet = global system of interconnected networks.
Big entity = LAN WAN MAN internet
WWW (world wide web) = Collection of websites aur content (photos, videos, audio, text).
IP address = Internet par har device ka unique address. Example: 192.168.1.1 (local network ke liye) ya public IP (global access ke liye).
Browser = tool jo web files, websites, aur web apps ko access karne ke liye use hota hai. Examples: Google Chrome, Opera, Microsoft Edge.
Browser engine = ek browser ka part jo webpage ko render karta hai. Example: Google Chrome ka engine "Blink" hai, Firefox ka "Gecko".
Client server model = client jo request karta hai (user ka browser). server jo data provide karta hai (jese Facebook ka server). example: jab tum www.fb.com search karte ho, client request karta hai aur server response bhejta hai.
Domain = Kisi website ka naam aur address. Example: google.com.
DNS (Domain Name System): Ye domain name ko IP address main convert karta hai, taki browser ko pata chale ki website kahan host hai.

Some extra notes about z-index: 
Agar ek element naye stacking context ke andar hai, toh uska z-index sirf usi context ke andar kaam karega.
Agar tumhe usko pure page ke upar lana hai, toh uska parent ka z-index bhi set karna padega!
1. z-index normally sirf stacking order change karta hai.
2. Agar z-index + position (relative, absolute, fixed) ho, toh naya stacking context create hota hai.
3. Ek stacking context ke andar waale elements sirf uske andar compete karte hain, pure page ke saath nahi!
4. Agar element dikh nahi raha ya overlap sahi nahi ho raha, toh dekhna padega ke koi naya stacking context toh nahi ban gaya.

Agar tumhe lag raha hai ke z-index kaam nahi kar raha, toh check karo:
✔ Uska parent stacking context bana raha hai kya?
✔ Agar bana raha hai, toh parent ka z-index bhi set karna padega!
Z-Index tabhi kaam karega jab uska stacking context sahi se set ho!✅

What is stacking context:
Stacking Context ek alag duniya hoti hai, jisme z-index wale elements apas mein organize hote hain.
Socho ek glass ka box hai, aur uske andar chhoti chhoti papers rakhe hain. Agar box ko upar neeche move karoge, toh uske andar ke papers ka order nahi badlega—sirf pura box move hoga!
Webpage par bhi aisa hi hota hai, Jab ek element new stacking context banata hai, toh uske andar ke saare elements ek alag "layer" mein chale jate hain.
Is layer ke andar jitne bhi z-index values hain, wo sirf isi context ke andar kaam karengi. Baahar ke elements is context ke andar waale elements ko directly affect nahi kar sakte!

Naya stacking context kab banta hai?
1. jab z-index diya ho (position bhi set ho)
2. Agar opacity < 1 ho
3. Agar transform, filter, ya perspective use karein
