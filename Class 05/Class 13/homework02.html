CSS Specificity

What is Specificity?

If there are two or more CSS rules that point to the same element, the selector with the highest specificity will "win", and its style declaration will be applied to that HTML element.

Think of specificity as a hierarchy that determines which style declaration is ultimately applied to an element.

Specificity Hierarchy

Every CSS selector has a position in the specificity hierarchy.

Priority Description
Inline style Highest priority
Id selectors Second highest priority
Classes and pseudo classes Third highest priority
Attributes Low priority
Elements and pseudo elements Lowest priority

More Specificity Rules Examples

Equal specificity: the latest rule wins - If the same rule is written twice into the external style sheet, then the latest rule wins.

ID selectors have a higher specificity than attribute selectors - Look at the following three code lines:

div#myDiv {background-color: green;}

#myDiv {background-color: yellow;}

div[id=myDiv] {background-color: blue;}

the first rule is more specific than the other two, and will therefore be applied.

Contextual selectors are more specific than a single element selector - The embedded style sheet is closer to the element to be styled.

Jab multiple rules ek element ke liye compete karte hain, toh:

Jo zyada specific hai (jaise contextual selector ya inline style), wo jeetegi.

Agar specificity same ho, toh closer rule (e.g., embedded style) apply hoga.

Contextual selector = Contextual selector ka use tab hota hai jab aap ek specific element ke andar ka koi child element target karna chahte ho.

Embedded CSS = Yeh directly HTML page ke <style> tag ke andar hoti hai.

A class selector beats any number of element selectors - a class selector such as .intro beats h1, p, div, etc

The universal selector (*) and inherited values - The universal selector (*) and inherited values do not impact the specificity weight value

iska mtlb ye ke universal selectors aur inherited values dono ki koi specificity value nhi hoti kisi bhi bari specificity waale rule se override hojaate hain

CSS inheritance ka matlab hai ke kuch properties (e.g., color, font-family) parent element se automatically child elements mein pass ho jati hain

Cascading Rule Priority:

Specificity: Rules with higher specificity override lower ones.

Location: Inline > Embedded > External.

Importance: !important overrides all.
